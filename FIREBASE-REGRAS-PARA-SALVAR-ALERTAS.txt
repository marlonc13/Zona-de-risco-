Se o botão ficar em "Salvando..." ou der erro de permissão, confira o Firebase:

1) Firebase Console > Firestore Database
- Crie/ative o banco Firestore.
- Use estas regras durante os testes:

rules_version = '2';
service cloud.firestore {
  match /databases/{database}/documents {
    match /marcacoes/{id} {
      allow read: if true;
      allow create: if request.auth != null
        && request.resource.data.userId == request.auth.uid
        && request.resource.data.comentario is string
        && request.resource.data.comentario.size() <= 300
        && request.resource.data.expiresAt > request.resource.data.createdAtMillis;
      allow update, delete: if request.auth != null && resource.data.userId == request.auth.uid;
    }
    match /{document=**} {
      allow read, write: if false;
    }
  }
}

2) Firebase Console > Authentication > Sign-in method
- Ative Email/Password.

3) Firebase Console > Storage
- Ative o Storage para fotos.
- Regras de teste para foto:

rules_version = '2';
service firebase.storage {
  match /b/{bucket}/o {
    match /alertas/{userId}/{fileName} {
      allow read: if true;
      allow write: if request.auth != null && request.auth.uid == userId;
    }
  }
}

Nesta versão também tem anti-spam no app:
- visitante não publica alerta;
- usuário logado só pode criar 1 alerta a cada 2 minutos;
- máximo de 5 alertas a cada 30 minutos por conta.
